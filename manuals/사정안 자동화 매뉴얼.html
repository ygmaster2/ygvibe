사정안 자동화 시스템

사용 매뉴얼

1. 개요

사정안 자동화 시스템은 학기말 사정안 작성 및 교과우수상 자료 수집 업무를 디지털화하여 담임교사·교과교사의 입력 작업과 관리자의 취합·출력 업무를 효율화하는 웹 기반 시스템입니다. 관리자·담임교사·교과교사 3개 역할로 나뉘어 있으며, Google Apps Script(GAS) 환경에서 구동되어 별도 설치 없이 브라우저로 바로 사용할 수 있습니다. 로그인 화면에서 "매뉴얼 새창으로 열기" 버튼을 누르면 언제든 이 매뉴얼을 확인할 수 있습니다.

2. 주요 기능

관리자전체 관리

  • 비밀번호 로그인(Google Sheets master 시트로 확인)
  • 시스템 설정 : 학년도, 현재 학기(1/2학기), 담임교사·과목교사 로그인 허용 여부를 설정하면 저장 즉시 전체 사용자에게 반영
  • 대시보드 : 학기별 담임 제출 현황(제출/임시저장/미제출)과 교과 업로드 현황을 반 · 과목 카드로 확인
  • 담임 화면 조회 및 인쇄, 학년별 전체 반 사정안 연속 출력
  • 데이터 전송 : 담임 자료를 양식_1학기/양식_2학기 시트로 정리
  • CSV 내보내기 : 학기별 교과우수상·표창 데이터를 CSV로 다운로드해 로컬 converter.exe로 한글(HWP) 문서 메일머지 생성 (CSV의 교과우수상 명단은 과목편제 시트에 등록된 순서를 따름)

담임교사학급 사정안 작성

  • 학년 · 반 · 이름 로그인(class_master 시트와 대조)
  • 1학기 : 재적 · 사정인원 · 출석일수 미달자 · 비고, 표창 대상자 5개 부문(모범 · 노력 · 선행 · 봉사 · 리더십) 입력
  • 2학기(학년말) : 1학기 항목 전체 + 개근자 명단(입력 시 인원수 자동 계산, 3학년은 "3년 개근자"로 자동 표시)
  • 임시 저장 · 최종 제출 · A4 가로 인쇄

교과교사교과우수상 파일 업로드

  • 학년 · 담당 과목(과목편제 시트 기준 자동 목록) · 이름 로그인
  • 나이스(NEIS) xlsx 파일 업로드 → 파일 내용(학년 · 과목명 · 학생 수) 확인 화면에서 검토 후 확정
  • 기존 업로드 데이터가 있으면 덮어쓰기 확인 후 재업로드
  • 학년 · 과목별 업로드 현황 확인

3. 사용 방법

관리자

  1. 관리자 로그인 후 시스템 설정에서 학년도 · 현재 학기 · 담임교사/과목교사 로그인 여부를 설정합니다.
  2. 대시보드에서 담임 제출 현황과 교과 업로드 현황을 확인합니다.
  3. 필요하면 반 카드 또는 "담임 데이터 조회"로 특정 반 사정안을 조회 · 인쇄하고, "학년별 전체 출력"으로 학년 전체를 한 번에 인쇄합니다.
  4. 담임 · 교과 자료가 모두 준비되면 "데이터 전송"으로 양식 시트를 채운 뒤, "CSV 내보내기"로 파일을 내려받습니다.
  5. 관리자 PC에서 converter.exe를 실행해 CSV를 불러오고 메일머지를 실행하면 한글(HWP) 문서가 생성됩니다.

담임교사

  1. 메인 화면에서 "담임 교사 로그인"을 누르고 학년 · 반 · 이름을 입력합니다.
  2. 1학기 탭에서 재적 현황과 표창 대상자 5개 부문을 입력합니다.
  3. 2학기로 전환된 경우 개근자 명단도 함께 입력합니다.
  4. 내용을 저장한 뒤 제출합니다. 제출 후에도 언제든 다시 수정 · 재제출할 수 있습니다.
  5. 필요하면 인쇄 버튼으로 A4 가로 형식으로 출력합니다.

교과교사

  1. 메인 화면에서 "과목교사 로그인"을 누르고 학년 · 담당 과목 · 이름을 입력합니다.
  2. 나이스에서 받은 xlsx 파일을 업로드 영역에 선택합니다.
  3. 파일 내용 확인 화면에서 학년 · 과목명 · 학생 수가 맞는지 직접 확인한 뒤 "업로드 확인"을 누릅니다.
  4. 기존 데이터가 있으면 덮어쓰기 여부를 확인하고 진행합니다.
  5. 업로드 현황에서 담당 과목이 정상 반영되었는지 확인합니다.

4. 코드 구조

Google Apps Script 프로젝트로 구성되며, code.gs(서버 로직)와 index.html(로그인 · 관리자 · 담임 · 교과교사 화면을 모두 담은 단일 페이지) 두 파일이 실제로 사용됩니다. code.gs가 class_master(담임 대조), student_list(반별 학생 명단), master(비밀번호 · 학기 · 학년도), 과목편제(과목 순서), 양식_1학기/양식_2학기(전송 결과) 등 여러 시트를 읽고 쓰며, index.html은 화면 전환(hidden 클래스 토글)과 검증 로직을 JavaScript로 처리합니다.

5. 주의사항

  • 담임교사 저장 · 제출 시 재적 · 사정인원 · 미달자, 표창 5개 부문(학생 · 사유 · 추천교사), (2학기는) 개근자 명단 중 하나라도 비어 있으면 오류가 발생합니다. 같은 학생을 두 개 이상의 표창 부문에 중복 선택해도 저장할 수 없습니다.
  • 표창 대상자 드롭다운은 student_list 시트 기준 반별 최대 35명까지만 인식됩니다.
  • 교과교사 업로드 시 과목명이 로그인 정보와 다르면 경고창이 뜨지만, 확인을 누르면 그대로 업로드가 진행됩니다. 학년은 자동으로 대조하지 않으므로 파일 내용 확인 화면에서 직접 확인해야 합니다.
  • 관리자 비밀번호는 master 시트 A2 셀에 저장되어 있으며, 분실 시 스프레드시트에서 직접 확인해야 합니다.
